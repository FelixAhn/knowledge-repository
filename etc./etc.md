### 프론트엔드 기술
```
- 프론트엔드 기술은 사용자가 직접 상호작용하는 웹 페이지의 UI를 구성하는 기술
- 주요 종류
  * HTML(Hypertext Markup Language): 웹 페이지의 구조와 내용을 정의하는 마크업 언어

  * CSS(Cascading Style Sheets): 웹 페이지의 스타일(레이아웃, 색상, 폰트 등)을 정의하는 스
                                 타일 시트 언어

  * JavaScript: 웹 페이지에 동적인 기능을 추가하는 스크립트 언어
    * DOM 조작 : JavaScript를 통해 HTML 요소를 동적으로 변경 가능
    * AJAX: 비동기적으로 서버와 데이터를 주고받아 페이지 새로고침 없이 콘텐츠를 업데이트 가능

  * 프레임워크 및 라이브러리
    * React: UI를 컴포넌트 기반으로 개발할 수 있는 JavaScript 라이브러리
    * Vue.js: 반응형 데이터 바인딩과 컴포넌트 기반 아키텍처를 제공하는 프론트엔드 프레임워크
    * Angular: TypeScript를 기반으로 한 웹 애플리케이션 프레임워크
```

### 백엔드 기술
```
- 백엔드 기술은 서버에서 데이터베이스와 비즈니스 로직을 처리하는 기술
- 주요 종류
  * 서버 언어
    * Node.js: JavaScript로 서버 사이드 개발을 할 수 있는 런타임 환경
    * Python: 간결하고 읽기 쉬운 문법을 가진 언어로, Django, Flask와 같은 웹 프레임워크를 
              제공
    * Java: Spring 프레임워크와 함께 대규모 애플리케이션에서 자주 사용
    * Ruby: Ruby on Rails라는 프레임워크로 유명하며, 생산성을 중시

  * 데이터베이스
    * RDBMS(관계형 데이터베이스 관리 시스템): MySQL, PostgreSQL, Oracle 등으로, 테이블 
                                      간의 관계를 통해 데이터를 관리
    * NoSQL: MongoDB, Cassandra 등으로, 비정형 데이터를 관리하며 스케일링에 유리

  * 웹 서버
    * Apache: 가장 널리 사용되는 웹 서버 소프트웨어
    * Nginx: 높은 성능과 가벼운 리소스 사용으로 인기 있는 웹 서버 및 리버스 프록시 소프트웨어

  * API(Application Programming Interface)
    * REST: HTTP 프로토콜을 기반으로 한 API 아키텍처 스타일
    * GraphQL: 클라이언트가 필요한 데이터만 요청할 수 있도록 설계된 쿼리 언어
    
  * 인증 및 보안
    * JWT(JSON Web Token): 사용자 인증을 위해 주로 사용되는 토큰 기반 인증 방식
    * OAuth: 외부 서비스와 인증을 연동하기 위한 프로토콜
    * SSL/TLS: 데이터 암호화를 위해 HTTPS 프로토콜에서 사용
```
