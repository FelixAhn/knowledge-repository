### HTTP
```
- HTTP 프로토콜 : HTTP(Hypertext Transfer Protocol)는 웹에서 클라이언트와 서버 간의 
                데이터를 주고받는 프로토콜
- HTTP 주요 특징
    * 요청/응답 모델 : 클라이언트가 서버에 요청을 보내고, 서버는 그에 대한 응답을 반환

    * Stateless : HTTP는 상태를 유지하지 않는 프로토콜로, 각 요청은 독립적이며 서버는 이전 요청
                  의 상태를 기억 x
    * HTTP 메서드
        * GET : 리소스를 요청할 때 사용
        * POST : 데이터를 서버에 제출할 때 사용
        * PUT : 서버의 리소스를 업데이트할 때 사용
        * DELETE : 서버에서 리소스를 삭제할 때 사용

    * HTTP 상태 코드 : 서버가 클라이언트의 요청을 처리한 후 반환하는 코드로, 성공(2xx), 
                     리다이렉션(3xx), 클라이언트 오류(4xx), 서버 오류(5xx) 등

    * URL(Uniform Resource Locator) : 웹 리소스의 위치를 가리키는 주소로, 프로토콜, 호스
                                      트, 경로, 쿼리 스트링 등으로 구성
```

### 웹 어플리케이션 동작 방식 예시
```
1. 사용자 요청 (프론트엔드[클라이언트] 측)
  - 사용자가 블로그 페이지에 접속하고, "새 게시물 작성" 버튼을 클릭
  - HTTP 요청: 사용자가 작성 페이지를 요청하면 브라우저는 서버에 HTTP GET 요청을 보냄. 
              예를 들어, GET /new-post와 같은 요청이 서버로 전송

2. 서버 응답 (백엔드 측)
  - 서버는 사용자의 요청을 받음. 서버는 요청된 페이지에 필요한 데이터를 데이터베이스에서 조회한
    후, HTML, CSS, JavaScript로 구성된 페이지를 생성하여 클라이언트에 응답
  - 서버 동작 : 서버는 /new-post 경로로 들어온 요청을 처리하기 위해 라우터를 사용
              서버는 필요한 데이터를 데이터베이스에서 조회(또는 초기화)하고, 이를 포함한 HTML
              파일을 생성, 서버는 HTTP 응답으로 HTML 페이지를 클라이언트에게 반환
  - 클라이언트 동작 : 브라우저는 서버로부터 받은 HTML, CSS, JavaScript를 렌더링하여 새 게시물
                  작성 페이지를 화면에 표시

3. 사용자 입력 및 제출 (프론트엔드[클라이언트] 측)
  - 사용자는 제목과 내용을 입력하고, "게시하기" 버튼을 클릭
  - HTTP 요청: 이때, 브라우저는 서버에 HTTP POST 요청 보냄. 예를 들어, POST /submit-post
              와 같은 요청이 서버로 전송. 이 요청에는 사용자가 입력한 제목과 내용이 포함

4. 서버에서 데이터 처리 (백엔드 측)
  - 서버는 POST 요청을 받아서 해당 데이터를 처리합니다.
  - 서버 동작: 서버는 클라이언트로부터 받은 데이터를 확인하고, 데이터베이스에 저장 데이터베이스에 성
             공적 으로 저장되면, 서버는 새로운 게시물의 ID와 함께 클라이언트에 응답 보냄

5. 사용자에게 결과 표시 (프론트엔드[클라이언트] 측)
  - 서버의 응답을 받은 클라이언트는 사용자가 작성한 게시물이 성공적으로 게시되었음을 알리는 페이지로 
    리다이렉션하거나, 성공 메시지를 화면에 표시
  - HTTP 응답: 서버는 새 게시물 페이지로 리다이렉트하는 HTTP 응답을 반환하거나, JavaScript를 
              사용해 성공 메시지를 표시하는 등 다양한 방식으로 클라이언트에게 결과 알릴 수 있음

6. 전체적인 흐름 요약
  - 사용자 요청: 클라이언트가 서버에 요청을 보냄 (GET 또는 POST).
  - 서버 처리: 서버가 요청을 받아 필요한 데이터 조회/처리 후 응답 생성.
  - 클라이언트 응답 처리: 클라이언트는 서버로부터 받은 응답을 화면에 렌더링하거나 다음 동작을 수행.
```
